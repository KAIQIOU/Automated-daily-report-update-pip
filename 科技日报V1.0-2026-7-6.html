RecursionError: maximum recursion depth exceeded while getting the repr of an object
掘金 (热门) ★★★★☆ 2026-07-05
为什么读
你写 Pydantic Schema 喂给 LLM 时，自引用类型可能让 SDK 静默炸递归——这篇是定位与修复的全套打法。
这篇讲什么
深入剖析 CPython 中 RecursionError 的三层保护机制（调用栈计数器、+50 最后防线、3.12+ 的 C 栈防护），并结合 Google GenAI SDK 真实案例讲解自引用 Pydantic 模型等五种生产级触发场景与排障流程。
对我们有啥用
对 vibecoder 团队强相关：用 Pydantic + Gemini/OpenAI 结构化输出时，自引用 Schema 极可能踩到这个坑；文章给了诊断链路和修复范式（visited set / 迭代改写），而非无脑调大限制。
关键要点
  • RecursionError 不只是计数器越界，CPython 还有 +50 层 Fatal Error 和 3.12+ 的 C 栈物理保护三层防线
  • 生产环境绝大多数 RecursionError 来自第三方库遍历自引用数据结构（如 Pydantic Self、ORM 双向关联、__repr__ 循环），而不是用户显式递归
  • 修复姿势是给递归算法加 visited set 或迭代改写，调大 sys.setrecursionlimit 是反模式（3.12+ 前会 segfault）
