这篇讲什么

对我们有啥用
对 vibecoder 是直接提醒:用 Claude Code 这类 agent 时不能只看 diff,要保持对代码的深层理解,否则后续迭代会被自己卡死。
关键要点
  • 核心概念是'认知债'(cognitive debt):你对代码的理解落后于代码本身的状态
  • '理解才能参与'——理解深度决定了你能否继续与 agent 创造性协作
  • agent 改的越多越复杂,人就越容易失去流畅思考所需的'概念库'
