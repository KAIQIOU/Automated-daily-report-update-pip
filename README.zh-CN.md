## Automated-daily-report-update-pip · 科技日报全自动更新

> 科技日报全自动更新 — 设计两个 agent 在定时任务上跑。一个采集原始信源,用 LLM 分析过滤生成结构化日报;另一个自动同步并按固定时间表投递到飞书群。

## 这是什么

一套**全自动运营的 AI 科技日报系统**:

- 每天上午 9:50 / 10:10 / 10:40 三段时间窗抓取 9 个一手信源(量子位 / 36 氪 / 掘金 / Simon 等)
- LLM 总结归类到 3 大主题(AI / Agent / 行业动态),每条**必保 4 字段**:星级 / 留言 / 发布日期 / 来源
- 渲染为单文件 HTML 报告
- 飞书 DM 推送 + Bitable 多维表格同步,失败有兜底

零人工干预,定时任务跑完即出报告,运营只看结果。

## 自动化架构

两个独立 agent,在 Windows `schtasks` 定时任务上跑:

```
[09:50 兜底] → [10:10 重试] → [10:40 终态]
   ↓
Agent 1: 数据采集
   - 9 信源 RSS / 抓取
   - 跨日去重(URL + 标题 + GH repo 三层 14 天滑动窗口)
   - LLM 总结归类(3 主题)
   - 单文件 HTML 渲染
   ↓
Agent 2: 飞书投递
   - 飞书群 DM 推送(规范命名"科技日报 YYYY-MM-DD")
   - Bitable 多维表格同步
   - 失败 retry + verify 双层
   ↓
报告落盘 + 飞书消息送达 = 0 人工
```

## 关键设计

| 设计点 | 怎么实现 | 为什么 |
|--------|----------|--------|
| 跨日去重 | URL + L2 title + GitHub repo 三层 14 天滑动窗口 | 实测去重 5/50 条(6-23→6-24) |
| LLM 兜底链 | 19→1 失败:timeout 60s + smart_truncate 智能截断 + markitdown 清旧 md + 重跑清 state.pushed | 防 9 信源中 1 个卡死 → 全天失败 |
| 反爬绕过 | curl 走 WinHTTP 代理挂 → Python urllib 禁代理 + UA + 2 次重试 | qbitai/36kr/juejin/simon 全部 OK |
| 必保字段 | 星级 / 留言 / 发布日期 / 来源 | 4 维信息密度高于"AI 总结" |
| 失败重试 | 3 段时间窗 9:50 / 10:10 / 10:40 + 双 verify 嵌套 | 单点失败不丢报告 |

## 怎么用

**做运营**: clone 本仓库 → 解压 `daily-digest-pipeline-skill-v1.5.zip` → 改 `config/sources.yml` 4 处(信源 URL / 飞书 webhook / Bitable app_token / 投递时间)→ 跑 `install_deps.bat` + `register_cron.bat` → 第二天的 9:50 就有第一份报告

**做审计**: 直接打开 `科技日报V1.0-*.html`,每份独立可读;对比不同日期看趋势

**做复现**: 改 README 同目录 `daily-digest-pipeline-skill-v1.5.zip`,看里面 `docs/` 5 份规范文档 + `scripts/` 全部源码
