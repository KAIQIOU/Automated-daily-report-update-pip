这篇讲什么

对我们有啥用

关键要点
  • 签名 Git 哈希并不唯一标识内容,攻击者可生成内容相同、签名合法、哈希不同的"孪生"提交
  • 三条攻击路径:ECDSA 代数求逆、OpenPGP 子包插入、CMS/SMIME DER 非规范重编码
  • 直接冲击 Nixpkgs/Go modules/GitHub Actions 等依赖钉死 commit hash 的生态,以及把 hash 当内容寻址主键的复现构建系统
