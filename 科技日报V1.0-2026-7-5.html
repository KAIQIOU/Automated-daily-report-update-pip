Phase 7.3 复盘：后台任务不只是“扔进队列”，还要能被看见
掘金 (热门) ★★★★☆ 2026-07-05
为什么读
做 AI 应用必踩『队列黑盒』坑，这篇给了可直接复制的观测范式和踩坑清单。
这篇讲什么
PrepMind AI 在 Phase 7.3 给后台任务加了可观测层：用 PostgreSQL 的 BackgroundJob 做权威状态、summary API 给前端轻量入口、进程内 EventBus 做失败隔离，避免队列黑盒。
对我们有啥用
对做 AI 应用的 vibecoder 来说，文档解析、embedding 等长任务几乎都会进队列；这套『任务表+summary+条件轮询+handler 隔离』的范式可直接复用，避免用户和开发者都摸黑。
关键要点
  • BackgroundJob 用 PG 做权威状态，只存脱敏 metadata，避免变成第二个内容库
  • activeCount 必须数据库真实统计，不能只看最近 N 条，否则旧卡死任务会被 UI 漏掉
  • EventBus handler 必须失败隔离，且日志不能 dump payload，防止敏感数据泄漏
